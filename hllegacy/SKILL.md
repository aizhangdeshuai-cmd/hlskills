# 旧项目首次分析流程

> 属于 `hlskills` 技能系统。只分析、不修改，目标是把旧项目彻底搞清楚。分析结论可衔接 `/hlrefactor`（重构）或直接用于日常维护。

## 通用纪律

- 只读分析，不修改任何代码
- 所有结论必须有 file:line 引用或命令输出证据
- 不确定的地方标注为 "待确认" 而非猜测

## 角色与 Agent 映射

| 角色 | Agent | 文件 |
|------|-------|------|
| 代码分析 | `architect` | `agents/architect.md` |
| 前端分析 | `architect` + `designer` | `agents/architect.md` + `agents/designer.md` |
| 安全分析 | `security-reviewer` | `agents/security-reviewer.md` |

## 分析流程

### 第一阶段：项目扫描（3 步）
1. **项目全景扫描** — `architect` 通读项目，输出：
   - 代码规模（总文件数、总行数、目录结构）
   - 编程语言与占比
   - 框架识别（前后端框架、ORM、构建工具）
2. **技术栈盘点** — 输出完整技术清单：
   - 语言/框架/库/数据库/中间件/工具链（含版本号）
   - 构建/部署/CI 配置
   - 第三方服务集成（支付/推送/存储/邮件等）
3. **依赖分析** — `architect` 检查：
   - 外部依赖清单（直接依赖 + 间接依赖）
   - 过期依赖（当前版本 vs 最新稳定版）
   - 已知漏洞依赖（CVE 编号、严重性）
   - 调用关系：哪些模块依赖哪些外部库

### 第二阶段：架构理解（3 步）
4. **模块划分分析** — `architect` 识别：
   - 每个模块的职责、边界、文件列表、代码规模
   - 模块间耦合度（高/中/低）
   - 识别核心路径模块（业务关键路径）
5. **数据流追踪** — `architect` 追踪完整请求链路：
   - 请求入口（路由/中间件/Controller）
   - 处理链（Service/Logic/Helper）
   - 数据层（Model/Repository/SQL/ORM）
   - 响应出口（View/JSON/文件）
6. **逻辑架构图** — 基于上述分析，输出：
   - 模块间调用关系图
   - 数据流向图
   - 标注关键路径与瓶颈模块

### 第三阶段：质量基线（3 步）
7. **代码质量基线** — `architect` 输出：
   - 测试覆盖率（如有）、测试框架、测试数量
   - 代码味道密度（按 `hlcode` 代码味道清单扫描）
   - 重复代码率
   - 复杂度分布（圈复杂度 Top 20 文件）
8. **安全基线** — `security-reviewer` 审查：
   - 依赖漏洞（`npm audit` / `pip-audit` 等）
   - 密钥泄露检查（硬编码密钥/Token/密码）
   - OWASP Top 10 风险扫描
   - 鉴权与授权机制审查
9. **数据库审计** — `architect` 分析：
   - 表清单、每表大致行数
   - 索引策略（有无缺失索引）
   - 迁移历史（迁移文件数量、最后的迁移时间）
   - 表关系（主外键、隐式关联）

### 第四阶段：知识沉淀（3 步）
10. **API 端点盘点** — `architect` 输出全部端点：
    - 路径、HTTP 方法、参数、返回值类型
    - 鉴权要求、速率限制
    - 调用方（前端哪个页面/外部哪个服务）

11. **前端设计规范提取** — `designer` 从代码 + 视觉中反推设计规范，**目标：新增页面时除文案不同外，所有视觉元素与当前系统一模一样**。

    > 📸 优先用模型原生多模态识别截图；不支持则用 Read 工具读取 PNG/JPG。如需深入分析视频，可使用可选外部 `videodb` 技能（需用户自行安装，未安装不影响本流程）。

    ### 提取方法
    - **代码优先**：读取前端源码（CSS/Tailwind 配置/组件源码/主题文件），提取精确数值
    - **视觉验证**：截图 → 测量 → 与代码提取值对照，确保准确

    ### 字体系统
    - 字体族（中文/英文/数字各自字体）、字号阶梯（h1~h6 / body / caption / small）、字重（light/regular/medium/bold）、行高、字间距
    - **代码来源**：Tailwind config / CSS 变量 / typography 组件

    ### 色彩体系
    - 主色系（primary 及其 hover/active/disabled 态）
    - 辅色系（secondary 各态）
    - 语义色（success / warning / error / info 各态）
    - 中性色（bg / border / text / disabled 各层级）
    - **代码来源**：CSS 变量 / Tailwind colors / 主题文件

    ### 控件规范（逐一从源码提取精确参数）
    - **按钮**：尺寸（large/default/small）、圆角、边框、填充、各状态色值（normal/hover/active/disabled/loading）、图标间距
    - **表格**：表头样式（bg/字体/对齐/排序图标）、行样式（奇偶行/hover/选中/行高）、边框/分割线、分页样式
    - **表单**：label 位置/字体、input/select/textarea 尺寸/圆角/边框/焦点态/错误态/disabled 态、必填标记、辅助文字
    - **筛选/搜索**：搜索框样式、筛选下拉/多选、筛选标签、日期选择器、工具栏布局
    - **弹框/抽屉**：宽度/高度/圆角/阴影/header/body/footer 布局、关闭按钮、遮罩层、动画
    - **卡片**：圆角/阴影/边框/padding/header/body 间距
    - **标签/Badge**：尺寸/圆角/颜色变体/可关闭态
    - **图标**：图标库、常用尺寸、与文字间距
    - **分割线/线条**：颜色/粗细/样式（实线/虚线）/间距

    ### 结构与布局
    - **页面级**：整体框架（侧边栏+顶栏+内容区）、侧边栏宽度/折叠态、顶栏高度/内容
    - **内容区**：标准内边距、最大宽度、栅格系统（列数/间距/断点）
    - **列表页**：搜索栏+表格+分页的标准布局间距
    - **表单页**：分组标题/字段排列/列数/间距
    - **详情页**：描述列表/标签-值对齐/Tab 切换

    ### 交互规范
    - 加载态：骨架屏样式/Spin 尺寸/进度条样式
    - 空态：空状态插图+文案+操作按钮
    - 错误态：Toast 位置/样式/持续时间、行内错误、页面级错误
    - 确认：Modal 确认/ Popconfirm / Drawer 确认的场景选择
    - 通知：Notification 位置/样式/关闭
    - 路由切换：页面过渡动画/loading

    ### 样式底层约定
    - CSS 方案（Tailwind/CSS Modules/Less/Sass/Styled Components）
    - 命名规范（BEM/camelCase/其他）
    - 覆盖第三方组件的策略（PrefixCls/StyleProvider/CSS 覆盖）
    - 响应式断点与各端的布局差异

    ### 输出格式
    - 一份独立的**前端设计规范文档**，保存到 `docs/design/spec.md`
    - 每个类别包含：代码证据（CSS 变量/源码片段）+ 视觉截图 + 精确数值表
    - 新增页面开发时，严格按此规范执行，保证视觉效果与现有系统完全一致

    ### 提取来源优先级
    1. **源码文件** → `tailwind.config.*` / `theme.ts` / CSS 变量文件 / 组件目录
    2. **页面截图** → 全部核心页面截图，标注每个元素的精确样式
    3. **已有文档** → 如有旧规范文档，对比源码提取差异

12. **文档差距分析** — `architect` 对比：
    - 现有文档清单 vs 应有文档清单
    - 标注缺失文档、过时文档、需要更新的文档

## 交付文档清单

旧项目分析交付时必须包含：

### 架构与技术
1. **项目技术栈清单** — 语言/框架/库/数据库/中间件/工具链及版本
2. **模块架构说明** — 模块职责、边界、规模、耦合度
3. **逻辑架构图** — 模块调用关系图 + 数据流向图
4. **ER 图与数据库设计** — 表结构、索引、迁移历史、数据量级

### 接口与前端
5. **API 端点清单** — 全量端点（路径/方法/参数/鉴权/调用方）
6. **前端设计规范文档** — 色彩/字体/间距/组件清单/布局/交互/样式约定

### 质量与风险
7. **代码质量基线报告** — 覆盖率/味道/重复率/复杂度
8. **安全基线报告** — 依赖漏洞/密钥泄露/OWASP 风险
9. **技术债清单** — 按严重性分级（严重/重要/次要），每项含修复建议
