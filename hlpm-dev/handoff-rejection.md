### 9. 实现偏离拒收（v10 新增，开发段发现 PRD 不符时填写）

**场景**：开发完成后发现代码与 PRD 不符（4 类场景，对应 `docs/consistency-matrix.md` 第 5 矩阵「代码实现追踪矩阵」的 4 种状态）

| 偏离类型 | 标识 | 示例 | 处理 |
|---------|------|------|------|
| **A 漏实现** | ❌ 未实现 | PRD "订单超时 30 分钟取消" 代码未实现 | 返回步骤 4 补齐 + 更新 `docs/consistency-matrix.md` 第 5 矩阵（清旧 SHA + 新 SHA + ✅）+ 重走步骤 7.5 |
| **B 错误实现** | ⚠️ 偏离（语义） | PRD "待支付→已取消" 代码只做"已关闭" | 返回步骤 4 重写 + 触发强同步 + 重走 7.5 |
| **C 非功能/安全漏实现** | ⚠️ 偏离（非功能） | PRD "bcrypt cost≥10" 代码用 MD5 | 返回步骤 4 重构 + 安全审查（步骤 9）+ 重走 7.5 |
| **D 范围越界** | ➕ 越界 | 实现"商品收藏"功能但 PRD 未定义 | 与产品段确认：保留则补 PRD / 移除则删代码 |

**每条偏离必填**：

| 字段 | 必填 | 说明 |
|------|------|------|
| 偏离类型 | ✅ | A / B / C / D 选一 |
| PRD 条款引用 | ✅ | 必须含 PRD 文件路径 + 章节 + 原文片段 |
| 实际实现描述 | ✅ | 代码实现的具体行为 |
| 修复计划 | ✅ | 步骤 + 责任人 + 预计时间 |
| 触发强同步 | ⬜ | 如涉及 B 类偏离，勾选（v5 规则：已通过评审不可豁免） |

**处理流程**：

1. 开发段发现偏离 → 立即更新 `docs/consistency-matrix.md` 第 5 矩阵（代码实现追踪矩阵）的"实现状态"列
2. 填写本拒收单 §9
3. 按偏离类型路由：
   - A → 返回步骤 4 补齐 → 步骤 7.5 走查 → 步骤 8
   - B → 返回步骤 4 重写 + 触发强同步 → 重走 7.5 + 5/7/9 评审
   - C → 返回步骤 4 重构 → 步骤 9 安全审查 → 重走 7.5
   - D → 与产品段确认 → 补 PRD 或删代码
4. 修复完成后必须更新 `docs/consistency-matrix.md` 第 5 矩阵的"实现状态"为 ✅（同时重新填 SHA）
5. 修复完成前不得进入步骤 8 分支完成

**与 v5 拒收原因分类的关系**：
- v5 4 类（缺失 / 不达标 / 内部矛盾 / 三项不一致）：**前置防御**（开发前）
- v10 §9 4 类（A/B/C/D）：**开发中/后防御**（开发中 + 完成后）
- 两表合起来 = 完整的"PRD 100% 实现保障"防御体系
