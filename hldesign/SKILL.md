# UI/UX 设计规范

## 角色

- **设计师**：`designer`
- **评审**：`designer` + `executor`

---

## 设计依据优先级

### 1. 有设计规范文档
严格遵循项目已有的设计规范文档（设计系统、组件库、色彩/字体/间距规范），`designer` 不得偏离既定规范。

### 2. 无设计规范文档
`designer` 必须**先向用户询问**以下信息后方可开始设计：

- **设计风格偏好** — 极简、科技感、可爱、企业级、Material Design 等
- **色彩倾向** — 主色调、是否有品牌色
- **参考案例** — 用户是否有喜欢的同类产品界面（可读取截图/设计稿，用 Read 工具分析视觉特征）
- **响应式需求** — PC端 / 移动端 / 平板适配范围
- **参考图片** — 如用户提供参考设计图/截图，使用 Read 工具读取，提取色彩/字体/布局/间距

---

## 设计原则

### 输出要求
- 设计稿必须输出为可预览的 HTML，用户可直接在浏览器中查看

### 交互状态覆盖
交互逻辑需在设计稿中体现全部状态：
- **hover态** — 鼠标悬停效果
- **点击态** — 按下/激活状态
- **加载态** — 数据加载中的骨架屏/loading
- **空态** — 无数据时的提示与引导
- **错误态** — 错误提示与重试入口

### 设计评审自查清单
设计评审前，`designer` 需自查：
- [ ] 是否覆盖所有交互状态
- [ ] 是否遵循设计规范
- [ ] 是否满足无障碍规范（WCAG 2.2 AA）

---

## 无障碍规范（WCAG 2.2 AA）

### 设计阶段必须满足
- 色彩对比度 >= 4.5:1（正文文本），>= 3:1（大文本/UI组件）
- 交互目标最小尺寸 24x24px
- 所有非文本内容提供文本替代
- 不仅依赖颜色传达信息（配合图标/文字/形状）

### 参考
完整无障碍规范见 `/hla11y` 命令。

---

## 设计与实现分离（🚨 DEV-NOT-FOR-PROD）

设计稿 = **真实页面** + **元信息面板**，两者**严格分离**。dev 拿到设计稿**只搬真实页面**，**禁止搬元信息到生产代码**。

### 元信息列表（禁止搬生产代码）

设计稿可能含以下元信息，**全部不进生产代码**：

| 类别 | 例子 | 为什么不能搬 |
|------|------|------|
| diff 三件套 | 区块 1（原页面代码 / diff 基线）+ 区块 2（改动清单） | 仅评审用，生产代码无意义 |
| 弹窗状态演示 | 区块 3.2-3.5（4 个弹窗的打开状态截图） | 仅评审用，生产代码按需弹窗 |
| 逻辑说明层 | `🔗 角标` + 抽屉 + `<script id="logic-data">` JSON | 设计稿内嵌的 PRD/TC 引用，运行时不需要 |
| 悬浮元信息面板 | 右下角 `📋 元信息` 按钮 + 展开面板 | 仅评审用，运行时是 UI 噪音 |
| `data-prd` / `data-state` / `data-tc` / `data-matrix` 属性 | 任何挂载的逻辑指针属性 | 元信息载体，运行时无用且污染 DOM |

### 生产代码搬运白名单

dev 拿到设计稿时，**只搬以下内容到生产代码**：

- ✅ **真实页面 DOM**：区块 3（含 3.1 默认 ACTIVE 状态）的所有元素
- ✅ **关联 CSS**：但**排除**以下类名/属性：
  - `.logic-*` / `.floating-meta-*`
  - `[data-prd]` / `[data-state]` / `[data-tc]` / `[data-matrix]`
  - `.logic-badge` / `#logicPanel` / `#floatingMetaToggle` / `#floatingMetaPanel` / `#logic-data`
- ❌ 不搬区块 1-2-3.2-3.5-4-5 任何元信息
- ❌ 不搬右下角悬浮框

### 自查清单

设计稿交付前，designer 需自查：
- [ ] 顶部有 `<!-- DEV-NOT-FOR-PROD -->` 注释 + 明确搬运白名单
- [ ] 区块 1-5 / 悬浮框外层有 `<!-- ⚠️ DEV-NOT-FOR-PROD -->` 围栏注释
- [ ] `data-*` 属性仅挂在真实页面元素上（元信息面板内部不算）
- [ ] 右下角悬浮框可正常展开/收起，不影响真实页面交互

dev 拿到设计稿时，需自查：
- [ ] 真实页面 DOM 完整可搬
- [ ] 元信息（区块 1-5 / 悬浮框 / `data-*` / `.logic-*` / `#floatingMeta*`）**已剥离**
- [ ] 运行时无 "📋 元信息" 按钮 / 🔗 角标 / logic-data 残留
