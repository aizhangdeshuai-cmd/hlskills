# hlquickstart — 5 分钟上手演示

> 属于 `hlskills` 技能系统。**专为第一次接触本仓库的用户设计**——跳过阅读全部 27 项子技能的 SKILL.md,先让你**实际跑一遍** AI 按流程做事的体验,再决定要不要深入。

> ## ⚠️ 这是演示,不是流程纪律
>
> 本演示的所有 mock 场景(虚拟 bug / 虚拟需求)都是**演练用**,**不会**修改你真实的项目文件,也不会触发 28/23/14 步完整流程。演示让用户感受到"AI 按文档一步步做"是什么体验,真实流程的纪律仍然**靠 Agent 自觉遵守**(详见各流程 SKILL.md 顶部的"流程纪律执行机制"警告)。

---

## 为什么需要这个 skill?

`hlskills` 仓库有 **27 项子技能 + 19 个 Agent**,每个子技能 SKILL.md 都是几十到几百行的详细流程文档。新用户读完 README 通常会问:

- "我应该先跑哪个 skill?"
- "跑起来是什么感觉? AI 真会按 28 步走吗?"
- "5 分钟内能不能给我一个'值回票价'的体验?"

`hlquickstart` 答案:**用 3 步演练让你 5 分钟内感受到**。

---

## 第 1 步:看一眼(30 秒)

```bash
# 确认 hlskills 已安装到 Claude Code
ls ~/.claude/skills/ | grep hl
# 应该看到 27 个 hl* 技能目录(含 hlquickstart 自身)

# 如果是空,先跑 hlsetup 安装
# /hlsetup
```

**预期输出**: 你应该看到 `hl-flow/` `hlpm/` `hldev/` `hlbug/` `hlquickstart/` 等 27 个目录(v16 后)。

如果数量不对或完全没有,先跑 `/hlsetup` 一键安装。

---

## 第 2 步:跑一个 mock bug 演示 hlbug(2 分钟)

**演示场景**: "项目里有个登录页报错,用户说点击登录按钮后白屏"。

启动方式: 在对话中说

```
按 hlquickstart 第 2 步演示:项目里有个登录页报错,
用户说点击登录按钮后白屏。请按 hlbug 流程走前 4 步给我看。
```

**预期行为**: AI 会加载 `hlbug/SKILL.md`,按 14 步流程的前 4 步走:

1. **加载已有文档** —— 跳过(无 `.hl/memory/`)
2. **Bug 分析** —— 用 `debugger` agent 角色扮演,描述"白屏可能原因:JS 错误 / 网络失败 / 路由错误 / 状态管理 bug"
3. **根因定位** —— 用 `architect` + `executor` 角色,提供"证据收集 → 根因分析 → 假设验证"四阶段框架
4. **修复方案** —— 给出 3 个候选方案,每个含预期工作量 + 风险

**演示意义**: 你会看到:
- AI **真的会**按文档一步步走(不会跳到第 14 步部署)
- AI 角色扮演**会用专业术语**(debugger 谈"regression isolation",architect 谈"调用栈")
- AI 会在关键节点停下问你("要选哪个方案?")

---

## 第 3 步:跑一个 mock 需求演示 hlpm 前 7 步(2.5 分钟)

**演示场景**: "产品经理让我加一个'订单列表页导出 CSV'按钮"。

启动方式: 在对话中说

```
按 hlquickstart 第 3 步演示:产品经理让我加一个
"订单列表页导出 CSV"按钮。请按 hlpm 流程走前 7 步(到设计稿确认)给我看。
```

**预期行为**: AI 会加载 `hlpm/SKILL.md`,按 28 步流程的**前 7 步**走:

1. **加载已有文档** —— 跳过(无 `.hl/memory/`)
2. **需求分析** —— 用 `analyst` 角色,梳理"导出 CSV"涉及的:筛选条件、分页、列选择、文件名规则
3. **竞品分析** —— 用 `AskUserQuestion` 问你"是否有参考产品?"
4. **PRD 编写** —— 生成一份 5-10 行的 mock PRD(不写完整 6 大模块,只为演示)
5. **PRD 评审** —— 5 方角色扮演(analyst/architect/designer/test-engineer/executor)各给 1 段意见
6. **UI/UX 设计** —— 生成一份 mock HTML 设计稿(灰度布局,只要 1 个区块)
7. **用户确认设计** —— **停在硬性关卡**,等你说"确认"

**演示意义**: 你会看到:
- AI 在第 2 步/第 5 步会**主动用 AskUserQuestion** 问(不是省略步骤)
- 第 5 步 PRD 评审会**真的产出 5 段不同意见**(不是泛泛"PRD 看起来不错")
- 第 7 步会**真的停下来等你确认**(不直接进入开发阶段)

**演示到这里就停**——**不会**自动进入第 8-23 步(开发/测试/部署)。剩下 16 步你读完 SKILL.md 自己决定要不要走完整。

## 演示结束后:5 分钟感受总结

跑完 3 步后,你的真实感受可能是以下几种之一:

| 感受 | 含义 | 下一步 |
|------|------|-------|
| "AI 真按文档走了,虽然 5 方会签像 5 种语气说话" | 你体验到了 hlskills 的核心 | 可以继续用 hlpm 跑真实需求(但不期望真·多 agent 进程级对抗) |
| "AI 没按文档走,跳过了第 5 步评审" | Agent 没自觉遵守纪律 | 手动打断让它重读 SKILL.md 第 5 步(详见 hlpm/SKILL.md 顶部警告) |
| "AI 跑得太死板,问太多问题" | 你不适合 23 步严格流程 | 用 `hlsetup` 装完后,试试只跑 `hlbug` 这种轻量流程(14 步) |
| "我想用 Codex / Cursor,这个演示跑不通" | hlskills 在其他平台是降级支持 | 详见 README.md 「平台支持分级」段 |

## 不做的事

`hlquickstart` 故意**不**演示的内容(避免新用户误以为 hlskills 能做):

- ❌ 不演示"5 方会签是真·多 agent 进程级对抗"——实际上是同 LLM 扮演
- ❌ 不演示"自动部署"——Claude Code 不能 SSH 到生产环境
- ❌ 不演示"硬性关卡 = Claude Code runtime hook"——实际靠 Agent 自觉
- ❌ 不演示"覆盖率 100% = 自动化校验"——核心模块靠 verifier + 人工签字

这些限制在 README 「⚠️ 能力边界声明」段写明,`hlquickstart` 不重复。
