# 架构决策记录（ADR）

---

## 何时记录 ADR

对以下类型的技术选型必须记录 ADR：
- **框架选择** — 前端框架、后端框架、ORM等
- **库选择** — 第三方库、工具链
- **架构模式** — 微服务/单体、六边形架构、事件驱动等
- **基础设施** — 数据库选型、消息队列、缓存方案
- **关键约定** — API版本策略、鉴权方式、部署策略

---

## ADR 格式

每个 ADR 文件存储在 `docs/adr/` 目录下，文件名格式：`NNNN-<slug>.md`

```markdown
# ADR-NNNN: <标题>

## 上下文
<描述当前技术背景、面临的问题、约束条件>

## 备选方案
1. <方案A> — <优缺点>
2. <方案B> — <优缺点>
3. <方案C> — <优缺点>

## 决策
<选择的方案及理由>

## 后果
### 正面影响
- <带来的好处>

### 负面影响
- <引入的限制或风险>

### 迁移计划
<如需迁移，描述步骤>
```

---

## 取代规则

当架构决策被新的决策取代时：
- 原 ADR 文件**不删除**，保留历史上下文
- 在原 ADR 状态中添加 "**已取代**"，并链接到替代者
- 新 ADR 中引用被取代的 ADR

```
## 状态
已取代 — 见 [ADR-0002: 采用 PostgreSQL 替代 MySQL](0002-postgresql.md)
```
