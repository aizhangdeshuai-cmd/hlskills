# 三项强同步纪律(核心约束)

> 本文件: `hlpm/disciplines/strong-sync.md`
> 适用范围: hlpm 全流程(产品段)
> 引用: SKILL.md §三项强同步纪律

---

## 总原则

**PRD / 设计稿 / 测试用例 三项必须逻辑完全一致:**
- 任一项因评审驳回而回退修改,**强制重走后续所有评审**(不可豁免)
- 三项版本号必须对齐(PRD vX = 设计 vX = 用例 vX)才能进入下一阶段
- 一致性矩阵是 9.5 步骤的单一可信源,开发段以此为验证基准
- 任何"评审驳回导致的内容变更"都触发强同步,包括字段级调整

---

## 强同步豁免:小改快通道

**触发条件**(全部满足才可豁免):
1. 驳回原因**仅涉及用词 / 格式 / 排版 / 示例值调整**,**不涉及业务规则、状态机、权限、字段定义、验收标准的实质变化**
2. 文档 diff < **20 行**(累计,不含空行 / 注释)
3. 版本号可不变(也可 N→N+1 自由选择)

**豁免流程**:
- `analyst` 在评审记录里**显式标注**: "本次修改为小改, diff < 20 行, 无实质业务/状态机/权限/字段变化"
- 后续评审的 reviewer 只需在评审意见里加一句: "确认无实质改动, 接受"
- 不重走完整评审流程, 直接进入下一阶段

**豁免不允许的情况**(命中任一 → 强制重走):
- 业务规则 R-N 增删 / 改条件
- 状态机状态 / 转移边变化
- 权限角色 / 矩阵变化
- 字段增删 / 类型 / 校验规则变化
- 验收标准条款变化
- 一致性矩阵任何 ✅ → ❌ 变化

> 豁免是**优化评审疲劳**的工具,**不是**绕过强同步的工具。`analyst` 误判豁免 → reviewer 应拒绝并要求重走。
