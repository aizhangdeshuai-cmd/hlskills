# 完成前验证 + 受阻停止 + 升级

> 本文件: `hlpm/disciplines/completion-validation.md`
> 适用范围: hlpm 全流程(也供 hldev / 其他技能参考)

---

## 完成前验证(硬性)

**任何阶段标记"完成"前,必须执行验证规则:**
- 识别能证明该阶段完成的验证命令,完整运行
- 读取完整输出并检查退出码,确认通过后方可声明完成
- **绝对禁止**: "应该通过了"、"看起来正确"、"可能没问题"
- **铁律**: 证据先于断言,无一例外

---

## 验证豁免清单

| 产出物类型 | 是否需命令验证 | 验证方式 |
| --- | --- | --- |
| 文档分析类(竞品分析 / PRD / 用户画像) | ❌ | `analyst` 内容自检 + grep 模糊字眼 |
| 设计稿 HTML | ❌ | `designer` 规范引用自检 + 浏览器渲染确认(走 `hlbrowse`) |
| 评审意见 | ❌ | reviewer 角色自检 + 模板覆盖 |
| 验收标准 / 非功能需求 | ❌ | 与 PRD 业务规则反推对齐 |
| 一致性矩阵 | ❌ | `verifier` 三方交叉比对 + ✅/❌ 判定 |
| 自检报告 | ❌ | `analyst` 逐项勾选 + 阻塞 9.5 通过 |
| **代码(产品段不写,本表仅供 hldev 段参考)** | ✅ | 测试 + lint + type check |
| **测试用例执行(开发段)** | ✅ | `pytest` / `jest` / `playwright` 退出码 |
| **部署 / 数据库迁移(开发段)** | ✅ | 部署命令 + 健康检查 + 迁移 dry-run |

---

## 程序化兜底:产品段每步后自检源代码

主 agent 在 1/2/3/4/5/6a/6b/7/8/9/9a/9.5/10/11 步骤完成后,**执行 `git status --short`**, 确认未产生 `.ts/.js/.py/.java/.go/.vue/.tsx/.jsx` 等源代码变更(允许 `.md` / `.html` / `.json` / `.yml` 配置文件)。

- 命中源代码变更 → **回退该变更 + 报警**("角色边界铁律违反:产品段不应改源代码"),并报告用户
- 例外: hlpm 阶段 6b 设计稿输出 `.html` 是允许的(本技能的"代码"指生产代码,非设计稿)

---

## 受阻停止纪律

- 任何步骤遇到阻塞,**立即停止**,不得猜测或强行绕过
- 向产品经理(`analyst`)报告阻塞情况
- 产品经理无法解决时,立即向用户提问
- **3 次修复/驳回尝试失败 = 质疑架构方案,升级到用户决策**
- **单项评审最多驳回 3 轮**,超出升级到用户决策

---

## 评审升级用户决策的呈现模板

**触发条件**: 同一份交付物(PRD/设计/用例)累计驳回 3 轮仍未通过。

**呈现模板**(`analyst` 用 AskUserQuestion 询问):

```text
⚠️ 已驳回 3 轮未达成共识,升级用户决策:

【交付物】PRD v3 / 设计 v3 / 用例 v3
【驳回 1 轮】<角色>: <理由 1 句话>
【驳回 2 轮】<角色>: <理由 1 句话>
【驳回 3 轮】<角色>: <理由 1 句话>
【当前卡点】<1 句话描述本质冲突,例如"业务规则 R-12 的边界条件定义">

请用户裁决(单选):
- A. 接受当前版本(暂时搁置争议)
- B. 继续修改(指定方向)
- C. 更换 reviewer(例如换人/换模型)
- D. 升级到集中评审(若当前是分阶段)/ 拆分为更小的子需求
```

> 升级后**不重置驳回计数** —— 同一个争议点累计 3 轮已证伪"内部可以解决",任何选项都需用户显式签字,不再自动继续。
