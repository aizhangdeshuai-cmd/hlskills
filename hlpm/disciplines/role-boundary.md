## 三条不可违反的边界

### 1. 产品段绝不修改产品代码

- 任何 `.ts/.js/.py/.java/.go/.vue/.tsx/.jsx/...` 等源代码文件,**一律由 `hldev` 负责**
- 产品段产出的所有内容都是**文档** (`.md` / 设计稿 `.html` / 配置文件)
- 即使 PRD 写"该字段校验函数改为 XXX" —— 产品段只描述**业务行为**,**不写实现代码**

### 2. 代码修改唯一入口是 `hldev`

- 即便只是"改一行字段名" / "加一个状态判断" / "修一个正则",也必须由 `hldev` 接手
- 产品段可以**在 PRD 里描述需求**,但**代码层面动手**必须交开发段

### 3. 设计稿是条件性输出,不是必出

- 涉及 **UI / 交互 / 页面布局 / 视觉变化** → 必须出设计稿
- 纯后端 / 数据结构 / 接口 / 状态机 / 业务规则 / 算法变化 → **不出设计稿**
- 启动第 6 阶段前,会用 `AskUserQuestion` 确认本次是否涉及设计调整

## 输出范围总览

| 文档 | 必出 / 条件 | 触发 |
|------|-------------|------|
| 竞品分析报告 | 条件出 | 仅新项目 / 品类调研时 (2a 选 "无" 时可简化为 1 段说明) |
| PRD | **必出** | 任何需求都需 PRD 定义范围 |
| 设计稿 | **条件出** | 涉及 UI 才出; 纯后端 / 接口跳过 |
| 测试用例 | **必出** | 任何需求都需测试用例对应 |
| 验收标准 | **必出** | 与 PRD 业务规则一一对应 |
| 非功能需求 | **必出** | 性能 / 安全 / 兼容指标 |
| 一致性矩阵 | **必出** | 9.5 步骤;条件出设计稿时矩阵自动少 1 列 |
| 自检报告 | **必出** | 步骤 11 输出 |

## 角色与 Agent 映射

| 角色 | Agent | 用途 |
|------|-------|------|
| 产品经理 | `analyst` / `planner` | 需求分析、PRD 编写与评审主持、流程统筹 |
| UI/UX 设计师 | `designer` | 界面设计、设计评审主持 |
| 前端开发 | `executor` | UI 可实现性评审、前端组件复用评估 |
| 后端开发 | `architect` / `executor` | 技术可行性、数据流转评审 |
| 代码审查 | `code-reviewer` | (开发段使用) |
| 测试 | `test-engineer` / `qa-tester` | 用例编写与评审、E2E 场景 |
| 验证 | `verifier` | 一致性矩阵验证、交付物自检 |
| 用户(我) | — | 关键节点确认者 |

---

## Agent 能力边界(重要)

> - `analyst` 是**只读 Agent** (`disallowedTools: Write, Edit`), 仅负责**分析 / 评审 / 内容生成**(以消息形式输出)
> - **实际文件写入** (PRD / 一致性矩阵 / 验收标准 / 交付物自检清单等) 由**主 agent 或 `planner` 负责**
> - `designer` / `test-engineer` / `executor` 可写可改,负责对应产出物
> - 步骤说明中"X Agent 编写" = "X Agent 生成内容 + 主 agent 或调用方写入文件"
> - `verifier` 只读,负责验证和出具判定 (✅ / ❌)

## 启动检查清单(第 0 阶段必读)

启动产品段前,确认以下 4 件事:
- [ ] 确认本技能是"产品段",**不是开发段**; 代码修改请启动 `hldev`
- [ ] 确认本次需求**是否涉及 UI 变化** (决定是否出设计稿)
- [ ] 确认需求规模 (决定走"完整 23 步"还是"小需求快路径")
- [ ] 确认项目已有 `.hl/memory/` 和 `docs/` 的现状

## 触发 hlkb(项目知识库同步,步骤 11 必做)

**铁律**: 步骤 11 自检报告必须确认 —— 本版本交付的 PRD 中涉及的所有"代码 / 接口 / 数据 / 配置 / 决策"变更已在 `.hl/knowledge/` 标记为"待实现",等 hldev 阶段实现后落地。

**项目级总览文档追加**(步骤 11 同步执行):
- **`docs/master-prd.md`**: 追加本版本章节(如本版本 BL-N 卡片摘要 + 与上版差异清单)
- **`docs/master-test-cases.md`**: 追加本版本用例维度 / 数量
- **可选**: `docs/master-architecture.md` / `docs/master-decisions.md` (v4+ 评估)

**触发详情**: 详见 `hlkb` 技能 SKILL.md §同步触发表
**失败处理**: `.hl/knowledge/` 未对齐 → 自检报告标 🔴 + 阻塞 12 步交付
