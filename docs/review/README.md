## 评审方法

每次评审都开 4 个独立的只读 sub agent,从不同角度并行探查;综合阶段合并去重(同一问题只记一次,标注来源视角数)。

## 状态约定

- ✅ **已修**:问题已解决,对应 commit 在 `git log` 中可查
- 🟡 **待修**:问题已识别,待下版本收口
- ❌ **放弃**:评估后决定不修(已在报告"放弃理由"段说明)
