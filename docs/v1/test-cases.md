## TC-001: 徽章存在性检查(主流程)

- **前置条件**:SKILL.md 文件存在
- **步骤**:
  1. 打开 `hlskills/SKILL.md`
  2. 读取前 10 行内容
- **预期结果**:
  - 至少存在 1 行 markdown 徽章(`![...](...)`)
  - 徽章含"v13"字样
  - 徽章含"3 合 1"或"联合评审"字样
- **验证命令**:
  ```bash
  head -10 SKILL.md | grep -E "^\!\[" 
  head -10 SKILL.md | grep -E "v13"
  head -10 SKILL.md | grep -E "3 合 1|联合评审"
  ```

## TC-002: 位置检查(异常流程-位置错误)

- **前置条件**:SKILL.md 文件存在
- **步骤**:
  1. 打开 SKILL.md
  2. 查找徽章所在行号
- **预期结果**:
  - 徽章必须在第 1-5 行内
- **验证命令**:
  ```bash
  line_no=$(grep -n "^\!\[" SKILL.md | head -1 | cut -d: -f1)
  test "$line_no" -le 5
  ```

## TC-003: 降级方案(异常流程-图片 404)

- **前置条件**:断网 / GitHub badges 不可达
- **步骤**:
  1. 用纯文字版徽章:`[hlpm-product v13] [评审模式: 联合评审 / 3 合 1]`
- **预期结果**:
  - 文字版徽章可读
  - 仍含版本号 + 评审模式信息
