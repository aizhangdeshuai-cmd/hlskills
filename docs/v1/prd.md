# PRD: hlskills 主入口 v13 状态徽章

> 项目: hlskills  
> 版本: v1  
> 创建日期: 2026-06-25  
> 评审模式: 3 合 1(轻量)  
> 路径: `docs/v1/prd.md`

## 1. 业务背景

hlskills 主入口 SKILL.md 当前没有版本状态标识,用户和开发无法一眼看出:
- 当前 hlpm 流程在哪个版本(v10 / v11 / v12 / v13)
- 当前默认评审模式是联合评审还是 3 合 1

加 1 行徽章解决以上 2 个信息透明问题。

## 2. 业务规则

| # | 规则 | 验证方法 |
|---|------|---------|
| BR-1 | 徽章显示 hlpm 当前版本号(v13) | `grep "v13" SKILL.md` 应有命中 |
| BR-2 | 徽章显示默认评审模式(联合评审 / 3 合 1) | `grep -E "联合评审\|3 合 1" SKILL.md` 命中 |
| BR-3 | 徽章位于 SKILL.md 顶部 5 行内 | 第 1-5 行应出现徽章 |

## 3. 操作流程

### 主流程
1. 用户打开 `hlskills/SKILL.md`
2. 在 description 字段后看到 1 行徽章
3. 鼠标 hover 看到版本号和评审模式

### 异常流程
- 如徽章图片 404 → 改用文字徽章(`[v13] [联合评审/3 合 1]`)
- 如 GitHub badges 服务不可用 → 退化方案:纯文字
